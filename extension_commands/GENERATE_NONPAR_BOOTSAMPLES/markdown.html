GENERATE NONPAR BOOTSAMPLES

GENERATE NONPAR BOOTSAMPLES B = value

  /OPTIONS OUTFILE = “filepath”

                  [SEED = value]

Description

GENERATE NONPAR BOOTSAMPLES generates a number of nonparametric bootstrapping samples for the active dataset. It is useful when users want to generate a set of nonparametric bootstrapping samples for further analysis (e.g., regression, reliability, correlation, etc.).

*B: This argument specifies the number of the bootstrap samples that the user wants to have. This is a required argument with no default.

*OUTFILE: This argument specifies the file name for the generated bootstrap samples. All samples will be stored in one single data file. It is suggested to use a full path for clarity. (e.g., "H:/temp/2iv_regression_bootsamples.sav"). This is a required argument with no default.

*SEED: This optional argument specifies an integer to be used as the seed for the random number generator. It can ensure that the result is reproducible.

The output data file will B x N cases, N being the number of cases in the dataset that is the active one when this command is run. The dataset will have B subsamples, identified by a variable b_i. A case id, boot_case_id, will be automatically generated. Each subsample will have N cases. All variables will be retained in the output data file.

Example

GENERATE NONPAR BOOTSAMPLES B = 5000

  /OPTIONS OUTFILE = “H:/temp/2iv_regression_bootsamples.sav”

                  SEED = 12345.

This example will create a new data file, "2iv_regression_bootsamples.sav", in the folder "H:/temp/". This file will have 5000 nonparametric bootstrap samples from the active dataset. The random seed is 12345. Every time this command is run on the same dataset, the same 5000 bootstrap samples will be drawn.

Custom Dialog

A custom dialog will also be installed along with this extension command. It can be launched by Generate Bootstrap Samples, under the Data main menu.. A video demonstration is available at https://osf.io/3bdmj/
